What is Quarto?

Quarto at a Glance

  • Quarto is an open-source scientific publishing system
  • Developed by Posit (formerly RStudio)
  • Write in Markdown, render to multiple formats:
    • PDF (via Typst or LaTeX)
    • HTML pages and websites
    • Presentation slides (like these!)
    • Word documents

Tip

Think of Quarto as a modern replacement for LaTeX that is simpler to learn, easier to use, and just as powerful for academic writing.

Why Quarto over LaTeX?

LaTeX

  • Steep learning curve
  • Complex syntax for simple tasks
  • Difficult to debug errors
  • Large installation (several GB)
  • Output mainly PDF

Quarto

  • Simple Markdown syntax
  • Easy to read and write
  • Clear error messages
  • Lightweight installation
  • PDF, HTML, Word, slides, websites

Note

Quarto produces publication-quality PDFs via Typst – no need to install a full TeX distribution!

LaTeX vs Quarto: Side-by-Side

LaTeX

\textbf{bold} and \textit{italic}

\begin{itemize}
  \item First item
  \item Second item
\end{itemize}

\section{Introduction}

\parencite{smith2023}

Quarto

**bold** and *italic*

- First item
- Second item

## Introduction

[@smith2023]

Same output quality – much less typing!

Or download the installer from quarto.org

Linux

Download the .deb or .rpm package from quarto.org

# Debian/Ubuntu
sudo dpkg -i quarto-*.deb

PDF Output with Typst

  • Quarto uses Typst for PDF rendering – it is built in!
  • No need to install a full TeX distribution (saves several GB)
  • Simply set format: typst in your YAML frontmatter
---
title: "My Paper"
format:
  typst:
    toc: true
    number-sections: true
---

Tip

Typst is fast, modern, and produces beautiful PDFs. It compiles in milliseconds rather than seconds.

Recommended Editors

  • Zed – fast, modern editor with built-in Quarto support
  • VS Code – install the Quarto extension for syntax highlighting, preview, and more
  • RStudio – has native Quarto support built in

Note

Any text editor works with Quarto files since they are just plain text (.qmd files). The editors above simply provide extra convenience features like live preview and autocompletion.

Quarto Basics

YAML Frontmatter

  • Every Quarto document starts with a YAML header between --- markers
  • Controls title, author, format, and options
---
title: "My Seminar Paper"
author: "Jane Doe"
date: today
bibliography: references.bib
format:
  typst:
    toc: true
    number-sections: true
---

Tip

YAML uses indentation (spaces, not tabs!) to define structure. Be careful with alignment.

Figures

  • Include images with captions and cross-reference labels
![Caption text](images/figure.png){#fig-example width="80%"}
  • Reference in text: @fig-example renders as “Figure 1”

Warning

The label must start with #fig- for Quarto to recognize it as a figure cross-reference.

Tables

  • Use pipe tables for simple data presentation
| Method | Objective | Time (s) |
|--------|----------:|:--------:|
| Exact  |   142.5   |   3.21   |
| Greedy |   138.7   |   0.05   |

: Comparison of solution methods {#tbl-results}
  • Reference in text: @tbl-results renders as “Table 1”

The label must start with #tbl- for table cross-references.

Math Equations

  • Inline math: $x^2 + y^2 = z^2$ renders as \(x^2 + y^2 = z^2\)
  • Display math with cross-reference:
$$
\min \sum_{i \in \mathcal{I}} \sum_{j \in \mathcal{J}} c_{ij} \cdot x_{ij}
$$ {#eq-objective}

\[ \min \sum_{i \in \mathcal{I}} \sum_{j \in \mathcal{J}} c_{ij} \cdot x_{ij} \tag{1}\]

  • Reference in text: @eq-objective renders as “Equation 1”

Academic Features

Citations with BibTeX

  • Store references in a .bib file and set bibliography: references.bib in YAML
  • Use the [@key] syntax to cite:
Syntax Output
[@smith2023] (Smith, 2023)
@smith2023 Smith (2023)
[@smith2023; @jones2024] (Smith, 2023; Jones, 2024)
[@smith2023, p. 42] (Smith, 2023, p. 42)

Tip

Use a .csl file (e.g., apa.csl) to control citation and bibliography style. Set csl: apa.csl in your YAML frontmatter.

Cross-References Overview

  • Quarto has a unified cross-referencing system:
Element Label Prefix Reference Syntax
Figure #fig- @fig-label
Table #tbl- @tbl-label
Equation #eq- @eq-label
Section #sec- @sec-label

Note

All cross-references are automatically numbered and updated when you add or remove elements. No more manual numbering!

How to Render

  • Render your document from the terminal:
quarto render paper.qmd
  • Or use live preview while writing:
quarto preview paper.qmd

Tip

quarto preview opens a browser window that automatically refreshes every time you save the file. This is the best way to write and check your formatting simultaneously.

Tips for Your Workflow

Reference Management

  • Use Zotero (free) with the Better BibTeX extension
  • Better BibTeX automatically exports your library to a .bib file
  • Workflow: find paper, save to Zotero, cite in Quarto with [@key]

Recommended Setup

  1. Install Zotero and the Better BibTeX plugin
  2. Set up auto-export to references.bib in your project folder
  3. Use the Zotero browser connector to save papers with one click
  4. Citation keys are generated automatically (e.g., smith2023)

Practical Tips

  • Keep figures in an images/ folder for clean project structure
  • Use quarto preview for live preview while writing
  • Write one sentence per line for better version control diffs
  • Use <!-- comments --> to leave notes for yourself

Key Takeaway

Quarto lets you focus on content, not formatting. Write your ideas in simple Markdown, and Quarto handles the rest – citations, cross-references, numbering, and beautiful PDF output.
